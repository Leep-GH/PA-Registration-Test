# PA Registry Tracker

A real-time tracker for France's certified e-invoicing platforms (Plateformes Agréées, PA) published by DGFiP (Direction Générale des Finances Publiques).

## Overview

**PA Registry Tracker** monitors the official [DGFiP list of certified platforms](https://www.impots.gouv.fr/je-consulte-la-liste-des-plateformes-agreees) and tracks changes over time. It detects when new platforms are added or registered, when platforms move to the certified list, and notifies subscribers via RSS feed and email.

### Key Features
- 🔍 **Real-time scraping** of DGFiP certified platform registry (daily schedule)
- 📊 **Historical tracking** of platform registration status changes
- 📧 **Email notifications** for new platform additions
- 📡 **RSS feed** for change notifications
- 🔐 **Secure subscriber management** with email confirmation
- 📱 **Public API** for programmatic access to platform data
- 🏃 **Dead man's switch** to alert on scraper failure (48-hour timeout)

## Tech Stack

- **Framework:** Next.js 14 (TypeScript)
- **Database:** PostgreSQL (production) / SQLite (development)
- **ORM:** Drizzle ORM
- **Web Scraping:** Cheerio
- **Email:** Resend
- **Testing:** Vitest 4.1
- **Hosting:** Vercel (frontend) + Supabase (database)
- **CI/CD:** GitHub Actions (daily cron, 3-retry, dead man's switch)

## Quick Start

### Prerequisites
- Node.js ≥ 20.0.0
- npm or yarn

### Local Development

1. **Clone the repository**
   ```bash
   git clone https://github.com/LeePetrie-Sage/PDP-Register.git
   cd PDP-Register
   npm install
   ```

2. **Set up environment variables**
   ```bash
   cp .env.example .env.local
   ```
   
   Then edit `.env.local` and set:
   - `ADMIN_SCRAPE_TOKEN`: Random secret for the `/admin/trigger-scrape` endpoint (generate a strong token)
   - `DATABASE_TYPE`: Set to `sqlite` for local development
   - `RESEND_API_KEY`: Resend email API key (get from [Resend](https://resend.com))

3. **Run the development server**
   ```bash
   npm run dev
   ```
   
   Open [http://localhost:3000](http://localhost:3000) in your browser.

4. **Run tests**
   ```bash
   npm test                # Run all tests
   npm run test:watch     # Watch mode
   npm run coverage       # Coverage report
   ```

### Database

The app uses Drizzle ORM with SQLite in development and PostgreSQL in production.

**Schema:**
- `pdps`: Certified platforms (name, URL, registration status)
- `change_events`: Tracked changes (platform changes, event type, timestamp)
- `scrape_runs`: Execution history (scraper runs, success/failure, platform count)
- `subscribers`: Email subscribers (email, subscription token, confirmed status)

**Migrations:**
Migrations are auto-generated by Drizzle. To create a new migration after schema updates:
```bash
npx drizzle-kit generate:sqlite
```

Then run: `npm run dev` (migrations apply on startup).

## Architecture

```
│                   DGFiP Public Website                  │

└──────────────────┬──────────────────────────────────────┘
                   │ Daily HTTP Request (GitHub Actions)
                   ▼
┌─────────────────────────────────────────────────────────┐
│                  Scraper Module                         │
│  (Cheerio parser → detect changes → store to DB)        │
└──────────────────┬──────────────────────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────┐
    │    PostgreSQL (Supabase)     │
    │  pdps / changes / runs /     │
    │       subscribers            │
    └──────────────────────────────┘
                   │
        ┌──────────┼──────────┬──────────┐
        │          │          │          │
        ▼          ▼          ▼          ▼
    ┌────┐   ┌────┐   ┌────┐   ┌────┐
    │API │   │Web │   │RSS │   │Email
    │Routes│ │Dashboard│ │Feed│  │Notify
    └────┘   └────┘   └────┘   └────┘
        │          │          │          │
        └──────────┼──────────┼──────────┘
                   │
                   ▼
            Deployed on Vercel
```

### Key Components

**Scraper** (`src/lib/scraper/`)
- `fetcher.ts`: HTTP fetch with retry logic and user-agent spoofing
- `parser.ts`: Cheerio HTML parser, extracts certified + candidate platforms
- `snapshot.ts`: Point-in-time snapshots for change detection
- `index.ts`: Orchestration, compares snapshots, logs changes

**Notifications** (`src/lib/notifications/`)
- `interface.ts`: Notification abstraction
- `console.ts`: Debug console output
- `rss.ts`: RSS feed generation
- Email: Handled via Resend API

**Database** (`src/lib/db/`)
- `schema.ts`: Drizzle schema definition
- `repositories/`: Data access layer (pdps, changes, runs, subscribers)

**API Routes** (`src/app/api/`)
- `v1/pdps`: List all platforms
- `v1/changes`: Paginated change history
- `v1/status`: Scraper health status
- `confirm`: Email confirmation endpoint
- `subscribe` / `unsubscribe`: Subscriber management
- `admin/trigger-scrape`: Manual trigger (requires `ADMIN_SCRAPE_TOKEN`)

**Frontend** (`src/app/`)
- **Dashboard** (`/`): Real-time stats, platform table, filtered search
- **History** (`/historique`): Change timeline
- **Detail** (`/pdp/[slug]`): Individual platform page
- **RSS** (`/rss.xml`): Machine-readable feed
- **Privacy** (`/privacy`): Privacy policy and data handling

## API Documentation

Full OpenAPI 3.x specification: [`docs/api/pdp-registry-tracker.yaml`](docs/api/pdp-registry-tracker.yaml)

### Example Requests

**List all certified platforms:**
```bash
curl https://pa-tracker.vercel.app/api/v1/pdps?status=certified
```

**Subscribe to email notifications:**
```bash
curl -X POST https://pa-tracker.vercel.app/api/subscribe \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

**Manually trigger scrape (admin only):**
```bash
curl -X POST https://pa-tracker.vercel.app/api/admin/trigger-scrape \
  -H "Authorization: Bearer YOUR_ADMIN_SCRAPE_TOKEN"
```

See [OpenAPI spec](docs/api/pdp-registry-tracker.yaml) for full endpoint documentation.

## Deployment

### Prerequisites
- Vercel account ([signup free](https://vercel.com/signup))
- Supabase account ([signup free](https://supabase.com/))
- GitHub repository connected to Vercel

### Deploy to Supabase + Vercel

1. **Create a Supabase project**
   - Go to [Supabase Dashboard](https://app.supabase.com/)
   - Create a new project (free tier)
   - Note the **Project URL** and **Anon Key**

2. **Run migrations on Supabase**
   ```bash
   npm run db:migrate
   ```
   
   This will apply `src/lib/db/migrations/0000_hot_firestar.sql` to your Supabase database.

3. **Set up Vercel deployment**
   ```bash
   npm install -g vercel
   vercel
   ```
   
   During setup, set these environment variables in Vercel:
   - `DATABASE_TYPE`: `postgres`
   - `DATABASE_URL`: Your Supabase connection string
   - `ADMIN_SCRAPE_TOKEN`: Strong random secret
   - `RESEND_API_KEY`: Resend email API key
   - `APP_URL`: Your deployed domain (e.g., `https://pa-tracker.vercel.app`)

4. **Configure GitHub Actions Secrets**
   
   Go to your GitHub repo → **Settings > Secrets and variables > Actions** and add:
   - `ADMIN_SCRAPE_TOKEN`: Same token as in Vercel
   - `APP_URL`: Your deployed domain

5. **Deploy**
   ```bash
   git push origin main
   ```
   
   Vercel will deploy automatically. GitHub Actions will run the scraper on schedule.

### Monitoring

- **Health endpoints:**
  - `GET /health/live`: Liveness probe
  - `GET /health/ready`: Readiness probe (checks DB connectivity)

- **Dead man's switch:** If the scraper fails to run within 48 hours, a webhook alert is sent. Configure the webhook URL in GitHub Actions environment variables.
